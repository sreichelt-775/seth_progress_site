Sensor behavior, data repair, and current sample equilibration plan.

Data quality issue and repair

The RH sensors showed multiple intervals where the signal deviated from the expected path. These deviations occurred at the same times in both RH% and RH temperature channels, which strongly suggests a DAQ/voltage-input issue. With 10-second sampling, this amounted to ~3,000 bad rows (~1.2% of the dataset).

I identified these intervals using an algorithm that flagged common-mode departures across all four RH sensors (coincident anomalies in RH% and RH temperature) relative to neighboring values. I repaired flagged segments using SciPy PCHIP interpolation with physical bounds consistent with typical sensor calibration tolerance. The repaired data returns to the prior trend with only mild variation.

I plan to pause LabVIEW, inspect the setup, and replace the power source to see if this recurs in future runs.
