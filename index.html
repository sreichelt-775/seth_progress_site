RH sensor issue and repair (tank-scale instrumentation)

The RH sensors showed multiple intervals where the signal deviated from the expected path. The deviations occurred at the same times in both the RH% and RH temperature channels, which strongly suggests a data-acquisition / voltage-input issue. With 10-second sampling, this amounted to ~3,000 bad rows (~1.2% of the dataset), identified by an algorithm that flagged common-mode departures relative to neighboring values, across all four RH sensors at the same timestamps.

I repaired the flagged segments using a interpolation in python (SciPy PCHIP), using physically reasonable bounds consistent with typical sensor calibration tolerance. This brought the data back onto the prior trend with only mild variation. The plots below show the RH% and RH temperature signals before and after repair, along with a zoomed overlay of raw vs. repaired data. I’m going to pause LabVIEW, inspect the setup, and replace the power source and see if this fixes the issue. I want to ensure the RH sensors are working before increasing the heater temperature. The highest temperature around the RH sensors is currently ~60°C.
