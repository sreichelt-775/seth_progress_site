RH sensor issue and repair (tank-scale instrumentation)

As i mentioned The RH sensors showed multiple intervals where the signal deviated from its expected path. These deviations occurred at the same times in both RH% and the RH temperature channels, which strongly suggests a data-acquisition/voltage input issue. With the 10-second sampling, this amounted to ~3,000 bad rows (~1.2% of the dataset) identified by an algorithm that flagged common-mode departures (across all 4 RH sensors sensors at the same time) relative to neighboring values.

I repaired the flagged segments using a python interpolation tool SciPy PCHIP with physical bounds matching the typical sensor calibration tolerance. This brought the data back onto the prior trend with only mild variation. fixed_Rh_data shows the repaired data and data_at_800_hours_RH_percent_repaired and data_at_800_hours_RH_temp_repaired show the full cleaned plot. I’m going to pause LabVIEW and inspect the setup and replace the powersource and see if this recurs in future runs.
